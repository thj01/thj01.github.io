Error handling

Murphy's lov siger:

Hvis noget kan gå galt, så vil det gå galt

Det betyder at man som programmør skal prøve at foregribe alle de fejl som fejlinput kan medføre. Det kan f.eks. være et decimaltal (float) der indtastes i et heltalsfelt (Int), eller bogstaver i et talfelt.

Se f.eks. nedenstående eksempel

	
a,b,c = 2,3,"d"

print(int(a))
print(int(b))
print(int(c))
	

2
3
Traceback (most recent call last):
  File "test.py", line 5, in 
    print(int(c))
ValueError: invalid literal for int() with base 10: 'd'

Python stopper kørseln af programmet fordi d ikke er et tal men en tekstreng, der ikke kan konverteres til et tal.
